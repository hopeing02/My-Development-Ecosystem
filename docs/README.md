# MDE 문서 체계

## 문서 흐름

`CON → GOV → STD → DEV → CMD → PRM → ARCH → SDS → ADR → QA → CODE → TEST → RELEASE`

## 디렉터리

- `01_constitution/` — **CON**: 헌법 · 최상위 원칙
- `02_governance/` — **GOV**: 거버넌스 · 역할과 변경 관리
- `03_standards/` — **STD**: 표준 · 문서와 명명 규칙
- `04_development/` — **DEV**: 개발 · 구현 규칙
- `05_commands/` — **CMD**: 명령 · AI 및 도구 명령 체계
- `06_prompts/` — **PRM**: 프롬프트 · 작성 표준
- `07_architecture/` — **ARCH**: 구조 · 시스템 아키텍처
- `08_sds/` — **SDS**: 상세설계 · SDS 색인 및 공통 기준
- `09_decisions/` — **ADR**: 설계 결정 · ADR
- `10_quality/` — **QA**: 품질 · QA 및 검증 기준
- `11_code/` — **CODE**: 구현 · 코드 기준 및 구현 이력
- `12_test/` — **TEST**: 테스트 · 계획과 결과
- `13_release/` — **RELEASE**: 릴리스 · 배포 및 변경 이력
- `14_operations/` — **OPS**: 운영 · 자동화, 워크플로, 에이전트
- `15_projects/` — 프로젝트별 SDS·설계·API·DB·UI·테스트·릴리스 문서
