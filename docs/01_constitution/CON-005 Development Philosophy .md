# 📘 CON-000

# 제5장 개발 철학 (Development Philosophy)

---

## 제23조 (개발 철학)

My Development Ecosystem(MDE)의 개발 철학은 다음과 같이 정의한다.

> **"설계하고, 표준화하고, 자동화하며, 지식으로 남긴다."**

개발은 단순히 프로그램을 만드는 과정이 아니라, 미래에도 재사용 가능한 자산을 구축하는 과정이다.

---

## 제24조 (개발 원칙)

MDE의 모든 프로젝트는 다음 10가지 개발 원칙을 따른다.

### 원칙 1. 설계 우선 (Design First)

구현 전에 설계를 완료한다.

최소 작성 문서

- 요구사항
- SDS
- 시스템 아키텍처
- 데이터 구조
- API 설계
- UI 설계

설계 없이 구현을 시작하지 않는다.

---

### 원칙 2. 표준 우선 (Standard First)

모든 프로젝트는 공통 표준을 따른다.

표준 대상

- 폴더 구조
- 파일명
- 문서 번호
- 네이밍 규칙
- 코드 스타일
- Git 브랜치
- 커밋 메시지

---

### 원칙 3. 문서와 코드의 동기화 (Documentation Sync)

문서와 실제 구현은 항상 일치해야 한다.

구현 변경 시 관련 문서를 함께 수정한다.

문서와 코드가 불일치하면 문서를 최신 상태로 갱신한다.

---

### 원칙 4. 작은 단위 개발 (Incremental Development)

프로젝트를 작은 단위로 나누어 개발한다.

예시

- 문서 1개
- 기능 1개
- 화면 1개
- API 1개

각 단위는 독립적으로 검토하고 테스트한다.

---

### 원칙 5. 테스트 중심 (Testable Development)

모든 기능은 테스트 가능해야 한다.

테스트 종류

- 단위 테스트
- 통합 테스트
- 사용자 테스트
- 회귀 테스트

---

### 원칙 6. 자동화 우선 (Automation First)

반복 가능한 작업은 자동화한다.

예시

- 문서 생성
- 코드 생성
- 테스트 실행
- Markdown 생성
- Git Commit 지원
- 릴리스 문서 생성

---

### 원칙 7. 재사용 우선 (Reuse Before Create)

새로운 기능을 만들기 전에 기존 자산을 검토한다.

검토 대상

- 코드
- 문서
- 프롬프트
- 템플릿
- 플러그인
- 워크플로

동일한 목적의 자산이 있다면 우선 재사용을 고려한다.

---

### 원칙 8. 품질 우선 (Quality First)

프로젝트 완료보다 품질을 우선한다.

품질 기준

- 가독성
- 유지보수성
- 안정성
- 확장성
- 테스트 완료
- 문서 최신화

---

### 원칙 9. 지속적 개선 (Continuous Evolution)

프로젝트는 완료로 끝나지 않는다.

프로젝트 종료 후

- 회고
- 개선 사항
- 오류 사례
- 재사용 가능한 자산

을 Knowledge Platform에 등록한다.

---

### 원칙 10. AI 협업 (AI Collaboration)

AI는 역할별 전문가(Agent)로 협업한다.

예시

- Planner Agent
- Architect Agent
- Developer Agent
- Reviewer Agent
- Tester Agent
- Documentation Agent

각 Agent는 자신의 역할과 책임을 수행하며, 최종 승인과 중요한 의사결정은 사용자가 담당한다.

---

## 제25조 (개발 라이프사이클)

모든 프로젝트는 다음 순서를 따른다.

↓

설계

↓

프로토타입

↓

구현

↓

테스트

↓

리뷰

↓

문서화

↓

배포

↓

회고

↓

지식 저장

↓

재사용

---

## 제26조 (완료 기준)

프로젝트는 다음 조건을 모두 만족할 때 완료로 간주한다.

□ 요구사항 구현 완료

□ 테스트 완료

□ 문서 최신화 완료

□ 코드 리뷰 완료

□ 변경 이력 기록

□ 지식 저장 완료

□ 재사용 자산 등록 완료

---

## 제27조 (개발 철학의 적용)

본 장의 개발 철학은 다음 문서의 상위 기준으로 사용한다.

- DEV-001 개발 표준
- STD-001 프로젝트 생성 표준
- PRM-001 프롬프트 표준
- CMD-001 AI 명령 체계
- CHK-001 품질 검수 표준

이들 문서는 본 철학을 구체적인 실행 규칙으로 확장한다.

✓ 개발 원칙 정의 완료

✓ 완료 기준 정의 완료

✓ 하위 문서와의 관계 정의 완료
