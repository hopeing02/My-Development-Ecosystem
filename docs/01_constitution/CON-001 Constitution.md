# 제1장 총칙

## 제1조 (명칭)

본 생태계의 공식 명칭은 **My Development Ecosystem(MDE)** 이다.

약칭은 **MDE**로 한다.

MDE는 개인이 AI와 함께 소프트웨어를 기획, 설계, 개발, 테스트, 배포, 문서화, 운영 및 지식 관리까지 수행하기 위한 통합 개발 생태계를 의미한다.

---

## 제2조 (목적)

MDE의 목적은 다음과 같다.

1. AI와 협력하는 표준 개발 환경을 구축한다.
2. 프로젝트의 모든 산출물을 지식 자산으로 축적한다.
3. 반복 가능한 개발 프로세스를 표준화한다.
4. 자동화를 통해 반복 작업을 최소화한다.
5. 장기간 유지 가능한 개인 개발 플랫폼을 구축한다.

---

## 제3조 (적용 범위)

본 헌법은 MDE에서 관리하는 모든 프로젝트에 적용한다.

적용 대상은 다음과 같다.

- Framework
- Core Engine
- AI Agent
- Workflow
- Plugin
- Knowledge
- Documentation
- Automation
- Project Builder
- 사용자 프로젝트

---

## 제4조 (기본 원칙)

MDE는 다음 원칙을 따른다.

### 1. Knowledge First

프로젝트보다 지식을 우선한다.

모든 프로젝트는 종료 후에도 재사용 가능한 지식 자산을 남겨야 한다.

---

### 2. Documentation First

모든 개발은 문서에서 시작한다.

문서 없이 개발을 시작하지 않는다.

---

### 3. Reusability

동일한 기능을 반복 구현하지 않는다.

문서, 코드, 프롬프트, 템플릿은 재사용 가능한 자산으로 관리한다.

---

### 4. Automation

반복적인 작업은 가능한 범위 내에서 자동화한다.

자동화는 사람의 판단을 보조하며, 중요한 의사결정은 사용자가 수행한다.

---

### 5. Continuous Improvement

모든 프로젝트는 이전 프로젝트보다 개선되어야 한다.

개선 사항은 문서와 지식 저장소에 기록한다.

---

## 제5조 (생태계 구성)

MDE는 다음 요소로 구성된다.

- Constitution
- Governance
- Standards
- Architecture
- Core Engine
- AI Engine
- Knowledge Platform
- Plugin System
- Project Builder
- Projects

각 요소는 독립적으로 관리되며, 표준 인터페이스를 통해 상호 연동한다.

---

## 제6조 (문서 우선 원칙)

다음 순서를 기본 개발 절차로 한다.

요구사항

↓

설계 문서

↓

구현

↓

테스트

↓

문서 보완

↓

배포

↓

지식 저장

모든 단계는 추적 가능해야 하며, 변경 이력이 관리되어야 한다.

---

## 제7조 (헌법의 우선순위)

CON-000은 MDE의 최상위 문서이다.

하위 문서는 본 헌법을 준수해야 하며, 충돌이 발생할 경우 CON-000을 우선 적용한다.

헌법 변경 시에는 관련 문서의 영향 분석과 개정 이력을 함께 기록한다.
