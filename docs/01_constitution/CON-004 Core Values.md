# 📘 CON-000

# 제4장 핵심 가치 (Core Values)

---

## 제20조 (핵심 가치)

My Development Ecosystem(MDE)는 다음 10가지 핵심 가치를 기반으로 설계되고 운영된다.

모든 프로젝트, 문서, 코드, AI 협업은 이 핵심 가치를 준수해야 한다.

---

# 09Core Value 01

## Knowledge First

지식은 가장 중요한 자산이다.

코드보다 오래 남는 것은 지식이며, 프로젝트보다 오래 남는 것은 경험이다.

모든 프로젝트는 다음 프로젝트를 위한 지식을 반드시 남겨야 한다.

대표 자산

- 설계 문서
- 요구사항
- 코드
- 테스트
- 프롬프트
- 오류 해결 사례
- 의사결정 기록

---

# Core Value 02

## Documentation First

문서는 개발의 결과물이 아니라 개발의 시작점이다.

모든 프로젝트는 다음 순서를 따른다.

요구사항

↓

설계

↓

개발

↓

테스트

↓

문서 보완

↓

배포

↓

지식화

---

# Core Value 03

## Standardization

표준은 품질을 만든다.

동일한 작업은 항상 동일한 방식으로 수행한다.

모든 프로젝트는 Framework 표준을 따른다.

예

- 폴더 구조
- 문서 번호
- API 규칙
- 네이밍 규칙
- Prompt 규칙

---

# Core Value 04

## Reusability

재사용 가능한 자산을 만든다.

같은 기능을 두 번 개발하지 않는다.

재사용 대상

- 코드
- 문서
- Prompt
- Workflow
- Template
- Plugin

---

# Core Value 05

## Automation

사람이 반복하는 일은 AI가 대신한다.

자동화 대상

- 문서 생성
- 코드 생성
- 리뷰
- 테스트
- Markdown 생성
- Git 반영
- Obsidian 정리

사람은 판단하고 AI는 실행을 지원한다.

---

# Core Value 06

## Continuous Improvement

모든 프로젝트는 이전 프로젝트보다 발전해야 한다.

개선 사항은 반드시 기록한다.

개선 대상

- 품질
- 속도
- 자동화
- 문서
- Prompt
- Workflow

---

# Core Value 07

## Modularity

모든 기능은 모듈화한다.

기능 간 의존성을 최소화한다.

새로운 기능은 Plugin으로 추가할 수 있도록 설계한다.

---

# Core Value 08

## AI Collaboration

AI는 도구가 아니라 협업 파트너이다.

AI는 다음 역할을 수행한다.

- 분석
- 설계
- 개발
- 리뷰
- 테스트
- 문서화
- 지식 정리

최종 결정은 사용자가 내린다.

---

# Core Value 09

## Long-Term Thinking

MDE는 단기 프로젝트를 위한 시스템이 아니다.

10년 이상 지속적으로 성장하는 생태계를 목표로 한다.

모든 설계는 장기적인 유지보수와 확장성을 고려한다.

---

# Core Value 10

## Human-Centered Development

기술은 사람을 위한 것이다.

AI는 사람의 창의성과 판단을 보조한다.

자동화는 사람을 대체하기 위한 것이 아니라, 사람이 더 가치 있는 일에 집중하도록 돕기 위한 것이다.

---

# 제21조 (핵심 가치 적용)

모든 프로젝트는 다음 사항을 검토한다.

□ Knowledge First를 준수하는가?

□ Documentation First를 준수하는가?

□ 표준을 준수하는가?

□ 재사용 가능한가?

□ 자동화 가능한가?

□ 지속적으로 개선 가능한가?

□ 모듈화되어 있는가?

□ AI 협업 구조를 따르는가?

□ 장기 유지보수가 가능한가?

□ 사람 중심의 설계인가?

프로젝트 검토 시 위 항목은 품질 평가 기준으로 사용한다.

---

# 제22조 (핵심 가치 우선순위)

핵심 가치 간 충돌이 발생하는 경우 다음 우선순위를 따른다.

1. Human-Centered Development
2. Knowledge First
3. Documentation First
4. Standardization
5. Reusability
6. Automation
7. Modularity
8. AI Collaboration
9. Continuous Improvement
10. Long-Term Thinking

이 우선순위는 설계와 구현 과정에서 의사결정의 기준으로 활용한다.
