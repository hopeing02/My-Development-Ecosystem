# 📘 GOV-001

# 제4장 의사결정 체계 (Decision-Making Framework)

---

## 제28조 (목적)

본 장은 MDE에서 이루어지는 모든 의사결정의 기준과 절차를 정의한다.

의사결정은 **투명성, 추적성, 책임성**을 원칙으로 한다.

---

## 제29조 (의사결정 원칙)

모든 의사결정은 다음 원칙을 따른다.

1. 최종 결정권은 User에게 있다.
2. AI는 분석과 제안을 담당한다.
3. 모든 중요한 결정은 기록한다.
4. 변경 사유는 문서화한다.
5. 동일한 기준을 반복 적용할 수 있어야 한다.
6. 가능한 경우 자동화하되, 승인 기준은 유지한다.

---

## 제30조 (의사결정 등급)

의사결정은 다음 네 단계로 구분한다.

### Level 1 : 자동 실행 (Automatic)

반복적이고 위험도가 낮은 작업

예시

- Markdown 생성
- 코드 포맷팅
- 문서 번호 생성
- 파일 정리

승인: 불필요

---

### Level 2 : AI 제안 (AI Proposal)

AI가 분석 후 권장안을 제시

예시

- 프로젝트 구조 제안
- 프롬프트 개선
- 코드 리팩터링 제안
- 테스트 케이스 생성

승인: User 확인 후 실행

---

### Level 3 : 공동 검토 (Collaborative Review)

영향도가 큰 설계 변경

예시

- Architecture 변경
- DB 구조 변경
- Workflow 변경
- AI Agent 추가

승인: User + 관련 검토 절차

---

### Level 4 : 전략 결정 (Strategic Decision)

생태계 전체에 영향을 미치는 결정

예시

- CON 개정
- GOV 개정
- 새로운 표준 추가
- 장기 로드맵 변경

승인: User 최종 승인

---

## 제32조 (AI의 권한)

AI는 다음 권한을 가진다.

가능

- &#x20;분석&#x20;
- &#x20;비교&#x20;
- &#x20;설계 제안&#x20;
- &#x20;코드 생성&#x20;
- &#x20;테스트 생성&#x20;
- &#x20;문서 작성&#x20;
- &#x20;품질 점검&#x20;

불가능

- &#x20;최종 승인&#x20;
- &#x20;정책 변경&#x20;
- &#x20;릴리스 확정&#x20;
- &#x20;헌법(CON) 개정&#x20;

---

## 제33조 (충돌 해결)

여러 Agent의 의견이 다를 경우 다음 순서를 따른다.

1. &#x20;근거 비교&#x20;
2. &#x20;품질 기준 적용&#x20;
3. &#x20;표준 문서 확인&#x20;
4. &#x20;User 판단&#x20;

충돌 내용은 Knowledge Base에 기록한다.

---

## 제34조 (RACI 적용)

모든 주요 작업은 RACI 원칙을 따른다.

| 역할의미 |       |
| ---- | ----- |
| R    | 실제 수행 |
| A    | 최종 책임 |
| C    | 협의    |
| I    | 공유    |

예시(SDS 작성)

| 역할담당 |                   |
| ---- | ----------------- |
| R    | Planner Agent     |
| A    | User              |
| C    | Architect Agent   |
| I    | Knowledge Manager |

의사결정 절차는 프로젝트 종료 후 회고를 통해 개선한다.

개선 사항은 다음 프로젝트에 반영하며, 필요 시 GOV 문서를 개정한다.
